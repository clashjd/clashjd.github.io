<h3>环境准备与基础工具配置</h3>

<p>要实现网络共享，首先需要确保你的主控设备（通常是手机）已经正确配置了代理客户端。这里我们以 Android 端的 Clash 和 iOS 端的小火箭（Shadowrocket）为例，演示基础的配置步骤。</p>

<h4>Clash for Android 配置步骤</h4>

<p>Clash for Android 是目前安卓平台上最主流的客户端之一。要开启共享功能，关键在于“覆写”设置。</p>

<ul>
    <li>首先，打开 APP，导入你的 <strong>Clash 订阅链接</strong> 或配置文件，确保当前手机可以正常访问目标网络。</li>
    <li>进入“设置” (Settings) 界面，找到“覆写” (Override) 选项。</li>
    <li>勾选 <strong>“允许局域网连接” (Allow LAN)</strong>。这是最关键的一步，默认情况下该选项通常是关闭的。</li>
    <li>为了防止端口冲突，建议检查“HTTP 代理端口”和“SOCKS5 代理端口”，默认为 7890 和 7891，通常无需修改。</li>
    <li>启动代理服务后，点击主界面的顶部栏，你会看到一行小字显示“局域网 IP：192.168.x.x”，请记下这个 IP 地址。</li>
</ul>

<h4>Shadowrocket (小火箭) 使用与配置</h4>

<p>对于 iOS 用户，<strong>Shadowrocket 使用</strong> 体验非常流畅，其共享逻辑更为直观。</p>

<ul>
    <li>在首页确保节点连接正常，开关处于开启状态。</li>
    <li>点击底部的“设置” (Settings)，找到“代理” (Proxy) 栏目下的“代理共享” (Proxy Share)。</li>
    <li>点击“启用共享” (Enable Share)。你会看到屏幕上显示了当前的 IP 地址和端口号（通常不需要手动设置端口，系统会自动分配）。</li>
    <li>值得注意的是，iOS 的热点机制较为特殊，有时需要保持屏幕常亮或 APP 在前台运行，以保证<strong>小火箭节点</strong>的持续转发。</li>
    <li>如果是通过 Wi-Fi 局域网共享，确保两台设备连接的是同一个 Wi-Fi；如果是通过个人热点，则连接手机热点的设备即处于同一局域网。</li>
</ul>

<h3>实用小工具与常见问题解答 (FAQ)</h3>

<p>在配置<strong>手机clash共享热点</strong>的过程中，很多用户会遇到连接超时或无网络的问题。以下是几个高频问题及解决方案。</p>

<h4>Q1: 接收端设备已经连接了 Wi-Fi，但无法上网怎么办？</h4>
<p>这是最常见的问题。通常是因为接收端设备没有手动配置代理。你需要进入接收设备的 Wi-Fi 设置，找到“代理”选项，选择“手动”，填入主控手机的 IP 地址（如 <code>192.168.43.1</code>）和端口号（如 <code>7890</code>）。</p>

<h4>Q2: 如何快速测试代理是否生效？</h4>
<p>在接收端设备的浏览器中访问查询 IP 的网站，或者直接使用 curl 命令测试连通性。例如：</p>
<p><code>curl -I https://www.google.com</code></p>
<p>如果返回 HTTP 200 或 301 状态码，说明连接成功。</p>

<h4>Q3: Clash for Windows 如何配合手机使用？</h4>
<p>如果你是在电脑上运行 <strong>Clash for Windows</strong> 并希望共享给手机，原理是一样的。在电脑端开启“Allow LAN”，鼠标悬停在 Allow LAN 选项上会显示电脑的局域网 IP。然后在手机 Wi-Fi 设置中填入该 IP 和端口（默认 7890）即可。</p>

<h4>Q4: Switch 或 PS5 连接总是失败？</h4>
<p>游戏机对 NAT 类型比较敏感。建议使用支持 UDP 转发的节点（如 Trojan 或 Hysteria 协议），并在 Clash 配置中开启 UDP 转发功能。此外，确保手机信号稳定，避免因移动网络波动导致断连。</p>
